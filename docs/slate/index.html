Dec 10, 2007
MPP 1.0.3 released, tested for MySQL Proxy integration, and contains mysql-polld.pl, a daemon-like replacement for cron polling.

MPP was released on its project host site, CodePin.cait.org, this year. Its first publicly written debut was in an article published on MySQL Dev Central November 1st. It is a three part article, part 1 and 2 are published, and the final part 3 is waiting to be published.

  • part 1 – MySQL Failover Strategy using State Management, introducing MPP - Part 1 (local archive)
  • part 2 – MySQL Failover Strategy using State Management, introducing MPP - Part 2 (local archive)
  • part 3 – MySQL Failover Strategy using State Management, introducing MPP - Part 3 (local archive)
Part 1 covers general terms, technology and methods for failover.
Part 2 covers more MPP specifics and implementation with LVS.
Part 3 covers MPP's general configuration, as well as its combination into MySQL Proxy completed in MPP release 1.0.3.

MPP is an Open Source project. The code is available freely at http://codepin.cait.org/project/mpp

MPP was born before MySQL Proxy was known, and I wrote it to plug into any load-balancer that can execute shell scripts or make REST calls to determine node status. As of version 1.0.3 it can be plugged into MySQL Proxy. Additionally, a working plan exists for the merger of MPP into the core of MySQL Proxy as Lua script.

Nov 13, 2007
MPP 1.0.1 released with support for MySQL Proxy integration.
